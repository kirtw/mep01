css^hgh(sel,parL)
    sel is the selector  with one of three simple formats
      .cls    pfx dot implies a class identifier
      #id    pfx # implies an id is used (must be unique in the page)
      No pfx implies a tag, ie an html element is being attributed.

    parL is a comma-delim (or _ delim) list of attribute triples
        each triple starts with the attr-name
         then a colon  (opt wsp) - (Here colon is analogous to equal-sign inline )
         and a value.
         The value may require a units designation without intervening wsp
         The value for short-cut property-names may have multiple space-delim parts.
        A semicolon ends the triple, but at the end is optionally added.

    An underscore may delimit a parL where spaces are embedded
        and would require quotes.
        Quotes are added and underscore replaced with space-delimiters for actual css.

    Single quotes are added to values automatically (?) I think...
